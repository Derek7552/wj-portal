# PRD功能模块拆解指令

## 拆解原则
1. **只关注功能设计，不包含视觉设计**
2. **模块化拆解，每个模块独立清晰**
3. **功能描述简洁明确，聚焦核心能力**
4. **避免过度设计，保持结构简单**

## 标准结构模板

```markdown
# [产品名称]需求文档

## 项目概述
**[产品/平台名称]：[核心定位描述]**

### 平台定位
[详细的产品定位和价值主张说明]

## 功能模块设计大纲

### 1. [模块名称]
#### 核心功能
- [功能点1：具体功能描述]
- [功能点2：具体功能描述]
- [功能点3：具体功能描述]

### 2. [模块名称]
#### 核心功能
- [功能点1：具体功能描述]
- [功能点2：具体功能描述]
- [功能点3：具体功能描述]

[继续添加其他模块...]
```

## 拆解步骤

1. **明确产品定位**：清晰描述产品的核心价值和目标用户
2. **识别核心模块**：将产品功能拆解为3-5个独立的功能模块
3. **定义核心功能**：每个模块列出3-6个核心功能点
4. **简化描述**：每个功能点用一句话清晰描述，避免冗长

## 注意事项

- ❌ 不要包含视觉设计、交互设计、界面布局等描述
- ❌ 不要包含技术实现细节
- ❌ 不要包含数据统计、性能指标等非功能需求
- ✅ 只关注"做什么"，不关注"怎么做"和"长什么样"
- ✅ 功能描述要具体可执行
- ✅ 保持模块间的独立性

## 示例

**好的功能描述：**
- 展示收录的行业安全资讯内容
- 资讯详情查看功能
- 支持分页或无限滚动加载

**不好的功能描述：**
- 卡片式布局展示资讯（包含视觉设计）
- 使用React组件实现列表渲染（包含技术实现）
- 响应式设计，支持移动端适配（包含视觉设计）
