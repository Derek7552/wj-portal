# UI界面设计要求拆解指令

## 设计原则

### 整体风格要求
1. **专业可信**：体现产品的专业性和技术感
2. **简洁清晰**：信息层次分明，避免视觉干扰
3. **现代科技**：采用现代化的设计语言，突出技术属性
4. **用户友好**：操作直观，降低学习成本

### 视觉规范要求
- 色彩搭配：符合产品定位的主色调
- 字体层级：清晰的信息层级，确保可读性
- 间距布局：合理的留白和间距，提升阅读体验
- 图标系统：统一的设计语言，增强识别度

```markdown
## SEO友好的界面设计

### 页面结构
- 使用语义化HTML标签
- 标题层级清晰
- 内容结构符合搜索引擎爬虫解析

### 内容展示
- 重要内容优先展示在页面可见区域
- 避免关键内容被JavaScript动态加载隐藏
- 图片使用alt属性，支持无障碍访问

### 链接设计
- 内链使用有意义的锚文本
- 相关推荐链接清晰可见
- 分类和标签链接便于搜索引擎发现

## 用户体验优化

### 加载体验
- 首屏内容快速加载
- 使用骨架屏或加载动画提升等待体验
- 图片懒加载，优化页面性能

### 交互反馈
- 按钮和链接提供明确的交互反馈
- 操作结果有明确的提示（成功、失败、加载中等）
- 错误状态有友好的提示信息

### 信息架构
- 信息层次清晰，重要信息优先展示
- 导航路径清晰，用户能快速找到目标内容
- 支持搜索功能，快速定位内容
```

## 拆解步骤

1. **明确设计原则**：根据产品定位确定整体风格和视觉规范
2. **设计页面结构**：规划全局导航和页面布局框架
3. **拆解功能模块**：为每个功能模块设计列表页、详情页和交互要求
4. **响应式适配**：定义不同屏幕尺寸的适配方案
5. **SEO优化**：考虑搜索引擎友好的设计要素
6. **用户体验优化**：关注加载、交互反馈和信息架构

## 注意事项

### ✅ 应该包含的内容
- 布局方式和信息展示要求
- 交互反馈和状态设计
- 响应式适配要求
- SEO友好的结构设计
- 用户体验优化考虑

### ❌ 不应该包含的内容
- 具体的像素值和尺寸（除非是必要的约束）
- 具体的颜色色值（只描述色调方向）
- 具体的技术实现方案
- 过于细节的视觉设计规范（如阴影、圆角等具体数值）

## 设计要点

### 布局设计
- 明确布局方式：卡片式、列表式、网格式等
- 信息层次：重要信息优先展示
- 视觉重点：关键操作按钮需要突出显示

### 交互设计
- 状态反馈：悬停、点击、加载等状态的视觉反馈
- 操作流程：用户操作的引导和流程设计
- 错误处理：友好的错误提示和状态展示

### 响应式设计
- 断点定义：桌面端、平板端、移动端的屏幕尺寸断点
- 布局适配：不同屏幕下的布局调整方案
- 交互适配：移动端的触摸操作优化

## 示例

**好的界面设计要求：**
- 卡片式布局，展示资讯标题、摘要、发布时间、标签
- 列表项支持悬停效果，提升交互反馈
- 行动召唤按钮需要突出显示，使用对比色或大尺寸设计

**不好的界面设计要求：**
- 使用16px字体，行高24px（过于具体的数值）
- 按钮使用#007AFF颜色（具体色值）
- 使用React的useState管理状态（技术实现细节）
