# ✍️ Clouditera 文案规范

> 提供文案设计原则、语言风格指南和常见场景示例

---

## 📐 核心原则

### 1. 从用户角度出发 👤
以用户为主体，关注用户能做什么，而非系统在做什么。

```
✅ 推荐：
- "你可以创建新的智能体"
- "查看任务详情"
- "开始分析"
- "保存配置"

❌ 不推荐：
- "我们为你提供了创建智能体的功能"
- "系统已为你保存"
- "我们的平台支持..."
```

### 2. 精简语句 ✂️
省略无用词汇，不重复用户已知事实。

```
✅ 推荐：
- "保存成功"
- "正在加载..."
- "删除智能体"
- "创建任务"

❌ 不推荐：
- "你的操作已经成功保存到系统中"
- "系统正在为你加载数据，请稍候"
- "你确定要删除这个智能体吗？删除后将无法恢复"
```

### 3. 使用用户熟悉的语言 💬
站在用户的角度，说用户熟悉的话。

```
✅ 推荐：
- "智能体"
- "任务"
- "知识库"
- "分析结果"

❌ 不推荐：
- "Agent"
- "Job"
- "Vector Store"
- "Analysis Output"
```

### 4. 表述一致 🔄
同一功能、同一概念在整个平台中保持一致的表达。

```
✅ 推荐：
统一使用 "智能体" 而不混用 "Agent"、"代理"
统一使用 "任务" 而不混用 "工作"、"作业"

❌ 不推荐：
页面 A 用 "智能体"，页面 B 用 "Agent"
```

### 5. 重要信息显著 ⭐
将关键信息放在显著位置，通过视觉层次突出。

```
✅ 推荐：
错误提示: "密码错误"（主要信息）+ "请重新输入"（次要信息）

❌ 不推荐：
错误提示: "很抱歉，系统检测到你输入的密码不正确，请检查后重新输入"
```

## 📝 语言风格

### 称谓
- 使用 **"你"** 称呼用户
- 使用 **"我们"** 仅在代表平台方时（如审核、处理申诉）

### 语气
- **友好**：使用温和、亲切的语言
- **专业**：使用准确的专业术语
- **正面**：多用肯定句，少用否定句
- **简洁**：避免冗长复杂的句子

### 时态
- 使用 **现在时** 描述当前状态
- 使用 **进行时** 描述正在进行的动作
- 使用 **完成时** 描述已完成的动作

```
✅ 推荐：
- "保存成功"（完成时）
- "正在加载..."（进行时）
- "文件大小超出限制"（现在时）
```

## 🎯 常见场景

### 1. 操作按钮

```
✅ 推荐：
- "创建"、"新建"、"添加"
- "保存"、"确定"、"提交"
- "删除"、"移除"
- "编辑"、"修改"
- "查看"、"详情"
- "取消"、"返回"
- "下一步"、"上一步"

❌ 不推荐：
- "创建新的"（"新的"多余）
- "保存配置信息"（"配置信息"多余）
- "确认删除"（按钮+确认框重复）
```

### 2. 成功提示

```
✅ 推荐：
- "保存成功"
- "创建成功"
- "删除成功"
- "上传成功"
- "发送成功"

❌ 不推荐：
- "恭喜！你的操作已成功完成"
- "操作成功，系统已为你保存"
```

### 3. 错误提示

**原则**：说明错误原因 + 提供解决方案

```
✅ 推荐：
- "文件大小超出限制（最大 10MB）"
- "密码错误，请重新输入"
- "网络连接失败，请检查网络后重试"
- "该名称已存在，请使用其他名称"

❌ 不推荐：
- "错误！"（没说明什么错误）
- "操作失败"（没说明原因和解决方案）
- "系统错误，错误代码：500"（技术性过强）
```

### 4. 确认对话框

```
✅ 推荐：
标题: "删除智能体"
内容: "删除后无法恢复，确定删除？"
按钮: ["删除", "取消"]

❌ 不推荐：
标题: "提示"
内容: "你确定要删除这个智能体吗？删除后将无法恢复，请谨慎操作。"
按钮: ["确定", "取消"]
```

### 5. 空状态

```
✅ 推荐：
- "暂无任务"
- "暂无数据"
- "还没有智能体，立即创建"

❌ 不推荐：
- "这里什么都没有"
- "数据加载失败"（应该说明是空还是加载失败）
```

### 6. 加载状态

```
✅ 推荐：
- "加载中..."
- "正在上传..."
- "正在分析..."
- "正在生成..."

❌ 不推荐：
- "请稍候..."（没说明在做什么）
- "系统正在处理你的请求"（过于冗长）
```

### 7. 表单标签

```
✅ 推荐：
- "智能体名称"
- "任务描述"
- "文件上传"
- "开始时间"

❌ 不推荐：
- "请输入智能体名称"（在 label 中重复）
- "名称："（缺少明确指示）
```

### 8. 占位符文本

```
✅ 推荐：
- "请输入智能体名称"
- "请输入任务描述，最多 500 字"
- "请选择日期"
- "搜索智能体或任务"

❌ 不推荐：
- "名称"（应该用指导性文字）
- "输入..."（不明确）
```

### 9. 帮助文本

```
✅ 推荐：
- "最多 50 个字符"
- "支持 .pdf、.docx、.txt 格式"
- "建议分辨率 1920x1080"

❌ 不推荐：
- "请注意字符数量限制"（没说明具体限制）
- "请上传正确格式的文件"（没说明支持的格式）
```

## ✅ 文案检查清单

写完文案后，请检查：

### 内容
- [ ] 使用"你"称谓（而非"我们"）
- [ ] 语言简洁明了（无冗余词汇）
- [ ] 术语使用一致
- [ ] 错误提示包含原因和解决方案
- [ ] 重要信息突出显示

### 语气
- [ ] 友好亲切
- [ ] 专业准确
- [ ] 正面积极

### 格式
- [ ] 标点符号正确
- [ ] 无错别字
- [ ] 大小写规范（专有名词）

---

## 🚫 避免事项

1. ❌ **过度客套**："非常感谢你使用我们的产品"
2. ❌ **技术术语**："API 调用失败，错误代码 500"
3. ❌ **责怪用户**："你输入的格式不对"
4. ❌ **模糊表述**："可能会失败"、"也许会..."
5. ❌ **过度使用感叹号**："保存成功！！！"
6. ❌ **中英混用**："创建新的 Agent"
